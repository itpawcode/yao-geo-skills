# 方法说明

`yao-geo-brand-graph` 的目标不是输出一张漂亮脑图，而是把品牌事实整理成可检索、可验证、可维护的实体关系层。

## 工作模型

| 层级 | 处理动作 | 输出 |
| --- | --- | --- |
| 来源层 | 记录来源类型、定位符、日期、事实主张和核验状态。 | 来源账本 |
| 提及层 | 抽取品牌名、产品名、人名、地点、案例、行业词和场景词。 | mention 列表 |
| 候选层 | 合并相似提及，保留别名、旧名、英文名和易混项。 | candidate 列表 |
| 规范层 | 选定 canonical entity，记录消歧理由和证据。 | 实体清单 |
| 关系层 | 抽取带方向、谓词、证据和可信等级的关系边。 | 关系清单 |
| 应用层 | 生成 Mermaid、JSON-LD、RDF 式三元组、测试场景和页面补强建议。 | 四件套报告 |

## 关系约束

- 关系必须有方向，例如 `品牌 -> 拥有 -> 产品`。
- 关系必须有证据 ID；没有证据的关系只能列为待确认或补强建议。
- 不把“领先”“高效”“创新”等营销形容词当作实体关系。
- 客户、合作伙伴、人物、授权数据和效果数字先过隐私检查，再进入图谱。
