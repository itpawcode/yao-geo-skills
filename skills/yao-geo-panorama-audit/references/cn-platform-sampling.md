# 国内平台采样规范

## 统一采样字段

| 字段 | 说明 |
|---|---|
| `sample_id` | 唯一样本编号。 |
| `platform` | DeepSeek、豆包、千问、Kimi、元宝。 |
| `prompt` | 原始问题，不改写成总结。 |
| `intent_type` | 推荐、比较、替代、教程、价格、风险、真实性、购买决策、场景解决。 |
| `sample_time` | 绝对时间，精确到分钟。 |
| `account_state` | 登录/未登录、会员状态、是否启用联网或搜索。 |
| `region` | 采样地域或网络环境。 |
| `answer_text` | 答案原文或可追溯摘录。 |
| `brand_mentioned` | 目标品牌是否出现。 |
| `brand_salience` | 高/中/低；判断品牌在答案中是核心对象、辅助提及还是边缘提及。 |
| `brand_recommended` | 是否被推荐或列入候选。 |
| `brand_rank` | 若有列表，记录位置。 |
| `competitors_mentioned` | 竞品列表。 |
| `cited_sources` | 答案显式引用来源。 |
| `search_sources` | 联网搜索出现的来源。 |
| `source_claim_match` | 引用是否支持对应断言：支持/部分支持/不支持/无引用。 |
| `citation_coverage` | 关键断言被引用支撑的覆盖程度：高/中/低。 |
| `fact_errors` | 错误描述。 |
| `missing_facts` | 缺失但应出现的关键事实。 |
| `answer_stability` | 多轮同题答案是否稳定：高/中/低。 |
| `query_variant` | 原题、口语改写、长尾场景改写或多轮追问。 |
| `confidence` | 高/中/低，依据采样轮次和证据稳定性。 |

## 平台侧重点

| 平台 | 额外关注 |
|---|---|
| DeepSeek | 结论稳定性、推理链条、答案组织方式、是否把推断写成事实。 |
| 豆包 | 短答案质量、口语问法覆盖、推荐理由是否清晰。 |
| 千问 | 联网来源、引用准确性、多源信息组织。 |
| Kimi | 长文本总结、来源引用、多轮追问后结论是否漂移。 |
| 元宝 | 微信公众号、视频号、腾讯生态信源和短问答表达。 |

## 采样要求

- 同一问题至少采样多轮；正式项目建议覆盖不同时间段。
- 每个平台使用同一组核心问题，必要时补平台口语化改写。
- 记录账号状态和联网状态，不把不同状态下的结果混算。
- 采样结论只用于基线和机会判断，不写成平台永久倾向。
- 品牌出现、品牌显著、品牌被推荐和品牌被引用必须分开记录；任何一项不能代替其它项。
- 引用源必须做断言级检查；只出现链接但不支持答案句子的，标为“引用错配”。
