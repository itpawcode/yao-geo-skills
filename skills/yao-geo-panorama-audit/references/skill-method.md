# GEO 全景诊断方法

## 能力边界

本 skill 只负责建立 GEO 基线、诊断差距并输出机会优先级。它不负责直接写最终文章、不负责页面重构实施、不负责后端归因实现，也不把一次 AI 答案采样当成稳定排名。

## 诊断对象

| 对象 | 需要回答的问题 |
|---|---|
| 品牌实体 | AI 是否知道品牌是谁、叫什么、提供什么、适合谁。 |
| 竞品格局 | AI 回答中有哪些竞品出现，品牌在同题下处于什么位置。 |
| 内容资产 | 官网、博客、FAQ、案例、白皮书、公众号、媒体稿是否能被抽取和引用。 |
| 页面结构 | 核心页面是否有结构化摘要、FAQ、表格、来源、实体一致性和可抓取文本。 |
| 外部信源 | 百科、媒体、社区、文档站、行业报告是否能支撑权威信号。 |
| AI 答案样本 | 品牌是否出现、是否被推荐、描述是否准确、引用是否可追溯。 |

## 研究化诊断口径

- 基本观测单位是“问题 x 平台 x 采样轮次 x 时间 x 账号状态 x 地域 x 答案 x 来源”。
- 可见性不是单一指标，必须拆成出现、实体显著性、推荐状态、推荐顺位、引用支持、事实准确性和答案稳定性。
- 引用不是装饰性字段，必须检查来源是否真的支持对应断言；不支持的引用要标为“引用错配”。
- 实体显著性不能只按品牌出现次数判断，还要看品牌是否位于答案核心位置、是否被解释、是否与场景问题形成关系。
- 多轮采样用于判断波动，不能把多轮结果平均成“确定排名”；报告必须保留样本明细和置信度。

## 执行步骤

1. 建立实体档案。
   - 统一品牌名、别名、英文名、产品名、服务、渠道、地区、资质、客户、案例和价格边界。
   - 同名品牌或代理商页面要先做消歧，不要把第三方页面当官方事实。
2. 生成问题样本。
   - 推荐型：某行业/场景有哪些方案值得选。
   - 比较型：品牌与核心竞品怎么选。
   - 替代型：有没有某竞品的替代方案。
   - 教程型：如何解决某业务问题。
   - 价格型：价格、版本、预算和性价比。
   - 风险型：是否靠谱、限制、合规、适用边界。
   - 真实性：品牌是否真实、是否官方、有哪些客户/案例。
   - 购买决策：什么情况下适合购买、谁应该参与评估、怎么做试用或采购。
   - 场景解决：围绕具体岗位、业务流程或行业痛点提出解决方案。
   - 必须输出问题覆盖矩阵，说明每类意图的样本数、平台覆盖和业务用途。
3. 平台采样。
   - 同题至少多轮，记录平台、时间、账号状态、地域、联网状态、Prompt 原文和答案原文。
   - 记录品牌出现、实体显著性、推荐位置、竞品出现、引用源、引用支持度、错误事实、缺失事实、答案稳定性和语气倾向。
4. 资产盘点。
   - 官网：首页、产品页、行业页、方案页、价格页、案例页、关于页、文档页。
   - 内容：文章、FAQ、白皮书、视频、公众号、媒体稿、百科、社区讨论、行业榜单。
   - 检查是否有答案优先段、事实表、FAQ、来源区、更新时间、作者/组织信息和内部链接。
   - 把品牌定义、产品边界、价格、客户案例、资质和效果数据拆成断言级事实卡，并标注来源。
5. 评分与诊断。
   - 使用 `geo-quality-model.md` 的 8 个维度评分。
   - 每个问题归类为事实问题、结构问题、信源问题或平台波动。
   - 低分原因必须落到可执行资产：页面、事实卡、来源、问题样本、采样设置或外部证据。
6. 机会地图。
   - 快赢修复：低成本修正错误事实、标题、FAQ、来源区。
   - 内容补齐：补高频问题、竞品对比、场景解决、风险说明。
   - 页面重构：重做结构、摘要、表格、Schema、内链。
   - 知识库补强：建立事实卡、禁用表达、来源索引。
   - 外部证据建设：补媒体、百科、行业报告、公开案例。
   - 监测闭环：建立 Prompt 库、采样表、复采周期和纠偏任务。

## 输出判断

- 如果缺少 AI 答案样本，可以输出低置信度诊断，但必须把平台采样列为 P0。
- 如果缺少官网或品牌资料，先做实体核验，不直接进入竞品结论。
- 如果用户只给一个 URL，仍可做轻量全景预诊断，但报告必须说明“未覆盖完整品牌资产”。

## 证据规则

- 官网事实、客户案例、价格、资质和数据必须给出来源与核验日期。
- 竞品信息没有来源时，只能作为采样观察或待确认假设。
- “最新”“第一”“最佳”“领先”“一定推荐”等表述必须有执行日证据，否则删除或降级。
- 平台答案只代表采样窗口，不代表长期稳定结论。
- 同一断言如果出现多个来源，优先记录官方来源、公开第三方来源和平台采样来源之间是否一致。
