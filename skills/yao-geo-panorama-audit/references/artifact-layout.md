# 全景诊断报告版式

## 报告结构

1. 封面：品牌、诊断范围、平台集合、采样窗口、版本日期。
2. 执行摘要：3-5 条关键发现、P0 动作、主要风险。
3. 方法依据与评分口径：说明黑箱采样、实体显著性、引用支持、事实准确和稳定性口径。
4. 问题样本覆盖矩阵：展示九类意图、样本数、平台覆盖和业务用途。
5. GEO 基线总览：品牌可见性、推荐情况、引用情况、事实偏差。
6. AI 答案可见性与竞品对比表。
7. 品牌实体与事实档案。
8. 官网与内容资产盘点。
9. 外部信源盘点。
10. GEO 特征评分。
11. 机会地图与优先级矩阵。
12. 页面与内容资产修复清单。
13. 附录：Prompt、采样记录、来源台账、待确认项。

## 版式原则

- 借鉴 `kami` 的专业文档纪律：清晰层级、紧凑信息密度、表格节奏、留白克制、证据优先。
- 本 skill 的报告一律使用白底背景：页面背景、正文承载区和打印背景都应为白色，不使用渐变、纹理、深色底、装饰性块面或花哨视觉元素。
- 面向管理层阅读，先结论后证据；执行摘要控制在一屏或一页内。
- 使用紧凑表格呈现对比、评分、来源、任务和优先级；表格边框、表头背景、列宽和数字对齐必须稳定。
- 每张密集表后补一句“判断含义”或“下一步动作”，避免只有表格没有解释。
- 不使用营销式大标题、空泛卡片和无来源截图；不要把章节做成漂浮卡片。
- 中文简体为默认语言；报告字段、模块名、状态、风险等级、动作标签必须使用中文简体。

## 四格式规范

- Markdown：作为内容母版或可追溯文本版，使用语义化标题、中文字段和标准 Markdown 表格；不放布局专用 HTML，不依赖外部图片才能读懂。
- HTML：生成独立 `.html` 或完整 HTML 包，白底、最大宽度受控、响应式表格可横向滚动或自动换行；设置稳定 `line-height`、`border-collapse`、单元格内边距和长词换行，避免文字、表格和边框溢出。
- PDF：优先从已校验 HTML 渲染，A4 页面、合理页边距、打印背景为白色；控制分页，避免表格、标题、脚注和附录被截断。
- Word：使用 `.docx`，保持与 Markdown/HTML 同一章节结构；应用标题样式、正文样式和表格边框，避免合并单元格造成错位或窄列文字挤压。
- Word 密集表格必须显式设置表格总宽、网格列宽、较小但可读的字号和单元格内边距；总宽应低于页面可用宽度，超过 6 列时优先拆成语义分组窄表，不允许依赖自动列宽或让表格贴边、向右溢出。
- 四种格式必须共用同一内容结构和同一组事实来源；不得在不同格式中出现指标、结论或优先级不一致。

## 自检与修复

- 交付前逐项检查 Word、PDF、HTML、Markdown 是否都存在且可打开。
- 检查 HTML 和 PDF 是否白底、无文字重叠、无表格溢出、无边框错位、无异常行距、无被截断页眉页脚。
- 检查 Word 是否保留标题层级、表格边框、中文字段和可复制文本；不要交付图片化整页报告。
- 检查 Markdown 是否可作为审阅母版，章节顺序与其它三种格式一致。
- 检查全部表头、字段、状态值和模块名称是否为中文简体；英文只允许保留品牌名、平台名、URL、缩写和用户原始素材。
- 发现任一格式排版或语言问题时，先修复源文件并重新生成四格式，再交付路径。

## 必要表格

- 平台采样表。
- 方法依据与评分口径表。
- 问题样本覆盖矩阵。
- 竞品出现与推荐对比表。
- 事实错误与缺失信息表。
- 官网/内容/外部信源资产表。
- GEO 特征评分表。
- P0/P1 机会地图。
- 页面与内容修复清单。
