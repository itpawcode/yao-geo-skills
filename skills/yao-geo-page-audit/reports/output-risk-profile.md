# 输出风险画像

- 高风险：把页面 GEO 诊断写成传统 SEO 文案建议，缺少 HTML 可读性和 AI 可抽取性。
- 高风险：schema 与正文事实不一致，导致结构化数据变成事实堆砌。
- 高风险：Word 宽表、长 URL、长命令向右溢出，导致交付体验差。
- 中风险：没有日志时误判爬虫真实召回。
- 中风险：只检查首页，忽略一级页、二级页、帮助中心、案例页和文档页。

## 缓解方式

- 使用五类诊断维度和 P0/P1 字段要求。
- 所有四件套从同一 Markdown 内容源生成。
- 使用 `scripts/review_report_layout.py` 检查 Word/PDF 右溢风险。
- 示例目录附带 `quality-report.json`。
