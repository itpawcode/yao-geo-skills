# GEO 页面蓝图质量门

- 首屏必须包含直接答案、结构化摘要和关键事实。
- 页面必须区分 `AI 可抽取模块` 与 `用户转化模块`。
- FAQ 不能只有底部堆叠；高价值问题应进入相关模块附近。
- 每个关键模块要写明可抽取字段、HTML 语义、来源和适配平台。
- Schema 不得包含正文没有的事实。
- CTA 不能干扰核心答案、证据区和比较表。
- 必须包含研究依据或方法依据映射，说明为什么这样安排页面结构。
- 必须包含证据区或来源台账，至少说明来源、核验日期、页面用途和相关模块。
- 必须检查 FAQ 是否真实出现在页面正文，且不是广告型问答；FAQPage 只能作为语义候选，不能写成 Google 富结果承诺。
- 必须交付 Word、PDF、HTML、Markdown 四种格式。
- 四格式必须共用同一内容结构、同一模块顺序和同一事实来源。
- HTML、PDF 和 Word 必须使用白底报告版式。
- Word 必须检查右侧溢出风险：表格宽度不能超过正文宽度，表格必须有 `tblGrid`、固定布局、单元格边距和段落 `wordWrap`；存在长 URL、英文产品名、Schema 字段或代码片段时必须软换行。
- 交付前必须完成自 review；发现格式缺失、字段英文化、内容不一致或排版问题时，先修复再输出路径。
