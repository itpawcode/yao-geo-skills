# 页面蓝图报告版式

- 借鉴 `kami` 的专业文档纪律：清晰层级、紧凑信息密度、表格节奏、留白克制、证据优先。
- 报告一律使用白底背景；不要用深色底、渐变、纹理、装饰性块面或过重阴影。
- 表格必须有边框、表头背景、稳定列宽、长文本换行和可读行距。
- HTML、PDF、Word、Markdown 必须共用同一内容结构和同一事实来源。
- Markdown 使用语义标题和标准表格。
- HTML 设置 `border-collapse`、`overflow-wrap`、响应式表格和 A4 打印。
- PDF 从已校验 HTML 渲染，白底、A4、页边距稳定。
- Word 保留标题层级、表格边框和可复制文本，不交付整页图片。
- Word 表格必须使用固定表格网格、显式总宽度、显式列宽、单元格内边距和段落断词；存在长 URL、英文产品名、Schema 字段和代码片段时必须插入软换行，防止向右溢出。
