# Artifact Layout

## 报告风格

- 中文简体，白底背景，正文优先。
- 不使用深色背景、渐变背景、装饰性光斑或复杂纹理。
- 标题层级清楚，正文行距稳定，表格边框对齐。
- 页面宽度受控，长 URL、英文产品名和长词必须自动换行。
- HTML、PDF、Word、Markdown 四格式共用同一内容结构。

## HTML/PDF 排版要求

- `body` 背景必须为 `#ffffff` 或等价白色。
- 表格使用 `border-collapse: collapse` 和 `table-layout: fixed`。
- 单元格使用 `word-break: break-word`、`overflow-wrap: anywhere`。
- 打印版避免元素跨页撕裂：关键表格、项目卡、路线图块使用 `break-inside: avoid`。
- 不使用 `gradient`、`rgba`、暗色大背景或悬浮卡片堆叠。

## Word 排版要求

- 必须包含标题、执行摘要、北极星指标、六个项目包、90 天路线图、平台动作、风险预案、来源台账和自检记录。
- 表格列数不宜过多；复杂信息拆成多个表。
- 标题与正文之间保持稳定段落间距。
- Word 表格必须使用固定页宽、固定列宽、`tblGrid`、`tcW` 和固定表格布局，禁止依赖自动表格宽度。
- 6 列以上且单元格为长句的内容不得直接输出为宽表；应拆成分平台、分项目包或分来源的两列表。
- 长 URL、英文产品名和连续英文词必须加入换行机会，避免把单元格向右撑出页面。

## Markdown 要求

- 标题层级从 `#`、`##`、`###` 顺序展开。
- 表格字段使用中文。
- 关键动作必须有负责人和验收指标。
