# Artifact Design Profile

## Artifact Family

Client-facing research report and reusable knowledge asset.

## Reading Goal

The reader should quickly understand which brand facts are safe to reuse, which facts need confirmation, and how the knowledge base can feed Chinese domestic AI platforms and downstream GEO production skills.

## Structure

- Executive summary
- Brand entity file
- Evidence tiering
- Brand fact-card library
- Reusable content modules
- FAQ and prohibited expressions
- Prompt input pack
- Competitor and industry context
- Pending-confirmation area
- Version and update mechanism
- Source index

## Visual Rules

- White background, A4 page geometry for Word/PDF.
- Dense information is acceptable, but tables must remain readable.
- Fact cards use five columns or fewer in visual formats.
- Source indexes with URLs use a two-column evidence ledger in visual formats.
- Long URLs keep full value but must wrap without pushing the page rightward.

## Acceptance Gates

- Four deliverables exist and are non-empty.
- HTML has fixed table layout and no local path leakage.
- Word passes fixed-table and long-token checks.
- PDF opens, preview PNGs are generated, and right-edge overflow scan passes.
