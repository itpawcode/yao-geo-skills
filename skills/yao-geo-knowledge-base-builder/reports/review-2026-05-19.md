# Review 2026-05-19

## Findings

1. `agents/interface.yaml` did not satisfy the portable interface contract used by `yao-meta-skill/scripts/validate_skill.py`.
2. `manifest.json` missed lifecycle fields: `updated_at`, `status`, `maturity_tier`, and `review_cadence`.
3. The renderer passed current examples but could silently pass zero-byte files because the existence check did not require non-empty files.
4. The renderer did not explicitly require PDF preview generation before the right-edge overflow scan.
5. Source-index compaction was Chinese-header-only; English or underscore-style source headers would not map reliably.
6. Wider fact-card tables were documented as disallowed but not automatically compacted.

## Repairs

- Rebuilt `agents/interface.yaml` with `interface` and `compatibility` sections.
- Added lifecycle fields to `manifest.json`.
- Added output-risk and artifact-design profiles.
- Hardened `scripts/render_four_format.py`:
  - file checks require non-empty output
  - PDF preview generation is a required quality check
  - source-index headers support Chinese and English variants
  - wide fact-card tables compact to the five-column visual contract
