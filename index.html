发布前要先讲清楚三件事

用户来到 GitHub 后，需要先判断这个仓库解决什么问题、自己该用哪个 skill、每个 skill 的输入输出和质量边界是什么。导航页负责快速浏览，README 和 docs 负责深入说明。

定位

说明 GEO 是 Generative Engine Optimization，仓库不是零散 prompt，而是可复用 skill 包。

选择

按战略、页面、内容、知识、监测、运营、研究分类，让用户知道该调用哪一个。

使用

每个 skill 都要有触发场景、必要输入、默认输出、示例和相邻边界。

信任

公开 eval、示例、质量门和开源安全规则，避免用户把未完成包当成稳定能力。
